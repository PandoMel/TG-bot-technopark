## 📁 Структура проекта TG-bot-propusk

```
TG-bot-propusk/
│
├── bot.py                    # ✅ Главный файл — Точка входа. Запускает бота, регистрирует роутеры и фоновые задачи (cleanup).
├── config.py                 # ✅ Настройки. Содержит токены, ID чатов, тексты сообщений и инициализирует объект bot.
├── database.py               # ✅ Хранилище. Отвечает за работу с bd.txt, хранит массивы данных в памяти и содержит функции поиска/записи.
├── services.py               # ✅ Бизнес-логика. Содержит вспомогательные функции: проверка прав доступа, таймауты сообщений, чтение "хвостов" логов.
├── logging_module.py         # ✅ Логирование. Настраивает ротацию логов и форматы записи в bot.log и KPP.log.
├── keyboards.py              # ✅ Интерфейс. Содержит все клавиатуры (Inline и Reply) и генераторы кнопок.
├── FSMstates.py              # ✅ Состояния. Определяет классы машины состояний (FSM) для сценариев диалога.
├── html_export.py            # ✅ Отчетность. Генерирует HTML-файл с пропусками(журнал пропусков) для визуального отображения на КПП.
│
├── routers/                  # ✅ Обработчики (Handlers)
│   ├── __init__.py
│   ├── user.py               # ✅ Обработка команд обычных пользователей (/start, заказ пропуска).
│   ├── admin.py              # ✅ Обработка админских команд (редактирование БД, просмотр логов).
│   └── events.py             # ✅ Обработка системных событий (вход/выход участников из чата).
│
├── bd.txt                    # База: "0\n123;ООО Ромашка;+7916..."
├── index.html                # Журнал пропусков (HTML)
├── bot.log                   # Лог бота (15MB rotating)
├── KPP.log                   # Лог пропусков (10MB rotating)
│
└── README.md                 # Этот файл
```

## 🔧 Как использовать
install Python 3.13

### 1. Установка зависимостей
```bash
pip install aiogram==3.14.0
```

### 2. Запуск бота
```bash
python bot.py
```


### Основные сущности (ID чатов)

Настройки лежат в `config.py`:
- `CHANNEL_ID` — “база доступа”: группа/супергруппа, по членству в которой бот разрешает функционал.
- `OHRANA_ID` — чат/группа/канал охраны, куда бот отправляет оформленные заявки на пропуск.
- `AXO_ID` — чат/группа/канал для заявок на ремонт (если включено).
- `ADMINS` — список user_id админов (для админ-меню и уведомлений).

### Настройка config.py

Заполните:
bot_token
ADMINS
CHANNEL_ID, OHRANA_ID, AXO_ID

## Настройка Telegram-чатов
Добавьте бота в CHANNEL_ID (группа/супергруппа) и сделайте администратором (это нужно для событий chat_member).
Добавьте бота в чат охраны OHRANA_ID (чтобы бот мог отправлять заявки).
Если используете заявки АХО/ремонт — добавьте бота в AXO_ID.

## 📌 Примечания

- База данных хранится в `bd.txt` в формате: `ID;Компания + ФИО;Телефон`.
- Файл читается при старте (load_bd()), дальше используется in-memory кэш.
- При изменениях кэш сбрасывается обратно в файл (write-through).

- Логи ротируются автоматически (15MB для bot.log, 10MB для KPP.log)
- HTML журнал обновляется при каждом новом пропуске
